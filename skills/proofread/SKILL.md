# /proofread : 한국어 교정

글의 **의미와 목소리는 건드리지 않고**, 어문 규범 오류만 바로잡는다. 산출물은 교정문 전문 + "무엇을 왜 고쳤는지" 내역이다.

> 이 스킬의 목표는 글을 더 멋지게 만드는 것이 아니라, **틀린 곳을 정확히 찾아 규칙대로 고치고, 그 근거를 남기는 것**이다.

## 제1원칙 : 의미 불변, 규범만

문장을 다시 쓰지 않는다. **표현·어순·문체는 그대로 두고 규범 오류만 고친다.** 더 나은 표현이 떠올라도 그건 교정이 아니라 재작성이다(그건 humanize의 일). 고친 곳은 전부 근거(규칙)를 남긴다.

## 절차 (이 순서를 지킨다)

1. **인터뷰** : 아래 [깊은 인터뷰]로 범위·표기 기준을 정한다.
2. **진단** : 규범 오류를 유형별로 찾는다 (맞춤법·띄어쓰기·문장부호·호응·외래어).
3. **교정** : 규칙에 따라 최소 수정한다. 의미가 바뀌면 멈추고 표시한다.
4. **대조** : 원문과 1:1 대조해 의미가 보존됐는지 확인한다.
5. **검증** : `scripts/verify.sh`로 흔한 오류 잔존·구두점을 확인한다.

## 깊은 인터뷰 (옵션은 고르게, 원문은 받는다)

**0단계 : 교정 범위 (선택 창으로 고르게 한다)**

- ① 규범 전체 : 맞춤법 + 띄어쓰기 + 문장부호 + 호응 + 외래어 (기본)
- ② 맞춤법·띄어쓰기만 : 빠른 오타 점검
- ③ 진단만 : 고치지 않고 오류 목록·규칙만 먼저 본다

**1단계 : 표기 기준**

- **문체 종결** : 합니다체 / 해요체 / 한다체 (혼용 통일 기준)
- **외래어 표기** : 국립국어원 외래어 표기법 기준 (예: 컨텐츠→콘텐츠, 메세지→메시지)
- **고유명사·전문용어** : 건드리지 말아야 할 표기 (브랜드명 등)

**(데이터)** 교정할 원문 : 직접 입력

## 교정 유형 (이 다섯을 본다)

| 유형 | 예 |
|------|-----|
| 맞춤법 | 되요→돼요, 몇일→며칠, 역활→역할, 오랫만→오랜만, 어떻해→어떻게 |
| 띄어쓰기 | 할수있다→할 수 있다, 그때→그때(붙임), 안 된다/안된다 구분 |
| 문장부호 | em-dash·물결표 오용→콜론, 따옴표·괄호 짝, 마침표 누락 |
| 조사·호응 | 주어-서술어 호응, 을/를·이/가, 시제 일치 |
| 외래어 표기 | 컨텐츠→콘텐츠, 메세지→메시지, 악세사리→액세서리 |

## 출력 형식

```
1. 교정문 전문 (의미·문체 그대로, 규범만 수정)

2. 교정 내역
   - (원문) → (수정) : (규칙·유형)
   - ...

3. 요약 : 유형별 건수 + 의미 변경 없음 확인
```

## 안티패턴 (이게 보이면 다시 한다)

- 더 나은 표현으로 문장을 다시 씀 → 그건 재작성이다. 규범만 고친다.
- 고친 곳에 근거(규칙)가 없음 → 교정은 근거가 생명이다.
- 고유명사·브랜드명을 임의로 바꿈 → 표기 기준에서 제외한 것은 건드리지 않는다.
- 의미가 달라졌는데 그냥 넘어감 → 의미가 바뀌면 멈추고 표시한다.
- 합니다체/해요체를 섞어 둠 → 종결 어미를 기준대로 통일한다.

## 검증 (필수 : loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <교정문.md> [원문.md]
```

- [ ] (자동·HARD) 흔한 맞춤법 오류 잔존 0건 (되요·몇일·역활 등)
- [ ] (자동·HARD) em-dash/en-dash 0건
- [ ] (자동·WARN) 흔한 띄어쓰기 오류 단서 (할수있다 등)
- [ ] (자동·WARN) 외래어 오표기 단서 (컨텐츠·메세지 등)
- [ ] (육안) 의미·문체가 원문과 동일
- [ ] (육안) 모든 수정에 규칙 근거가 있음
- [ ] (육안) 종결 어미가 일관됨

## 체인 예시

```
(초안)  →  proofread  →  humanize(공식결)  →  발행
report  →  proofread  →  공유
email   →  proofread  →  전송
```
