# 보고서 결 : 구조와 목소리를 고른다

인터뷰 0단계에서 아래 **보고서 결**을 선택 창으로 제시하고, 고른 결의 템플릿을 복제한다. 자유 입력이 아니라 **고르게** 한다.

기존엔 "보고서 종류(분석/현황/조사/제안)"를 고르고 하나의 템플릿으로 만들었다. 그건 구조만 달랐고 시각 언어는 같았다. 그래서 종류를 **결 안으로 흡수**했다. 결은 종류만이 아니라 **섹션 순서·강조 요소·시각 언어·독자 가정**까지 통째로 정의한다. 결이 다르면 같은 데이터도 전혀 다른 보고서가 된다.

---

## 결 3종

### ① 임원 요약결 (Executive Brief)
`templates/report.executive.html.txt`

| 축 | 값 |
|----|----|
| **독자** | 의사결정자 (시간 없음, 결론만 필요) |
| **목적** | 결정을 이끄는 것 |
| **섹션 순서** | 한 줄 결론 → 핵심 수치 3개 → 권고 행동 → 근거 요약(선택) |
| **밀도** | 1~2 스크롤 이내. 모든 내용이 위에 있어야 함. |
| **강조** | 수치 크게, 권고 행동 박스 강조, 배경 설명 최소화 |
| **어조** | 단정·직접. "검토가 필요합니다" → "3분기 예산 삭감을 권고합니다" |
| **언제** | 경영진 보고, 투자자 업데이트, 이사회 자료, 1페이지 요약 |

**섹션 체크리스트:**
- [필수] 한 줄 결론 (첫 번째 섹션)
- [필수] 핵심 수치 KPI (최대 3개)
- [필수] 권고 행동 또는 요청 (마지막 섹션)
- [선택] 배경·맥락 (짧게, 결론 뒤에)
- [선택] 리스크 1~2줄

---

### ② 분석 보고결 (Deep Analysis)
`templates/report.analysis.html.txt`

| 축 | 값 |
|----|----|
| **독자** | 실무자·팀 (근거가 필요, 상세 분석 원함) |
| **목적** | 이해와 설득 |
| **섹션 순서** | 요약(결론) → 배경·맥락 → 데이터·분석 → 인사이트 → 한계·주의 → 권고 |
| **밀도** | 충분히 긴 문서 허용. 수치 출처 전부 명시. |
| **강조** | 데이터 표, 수치 대조, 인사이트 박스, 출처·한계 각주 |
| **어조** | 근거 기반·논리적. 주장은 데이터로 뒷받침. |
| **언제** | 시장 조사, A/B 테스트 결과, 전략 분석, 경쟁사 벤치마크 |

**섹션 체크리스트:**
- [필수] 요약 (결론 먼저, 상세 분석 앞에)
- [필수] 데이터·분석 섹션
- [필수] 인사이트 (데이터에서 무엇을 읽었나)
- [필수] 한계·주의 사항 (모든 분석의 신뢰성 기준)
- [권고] 출처·방법론

---

### ③ 현황 보고결 (Status Update)
`templates/report.status.html.txt`

| 축 | 값 |
|----|----|
| **독자** | 프로젝트 관계자·팀원 (빠른 스캔, 이슈 파악) |
| **목적** | 투명한 현황 공유 |
| **섹션 순서** | 현황 요약(상태 배지) → 이번 기간 진행 → 이슈·리스크 → 조치 계획 → 일정·마일스톤 |
| **밀도** | 핵심만. 이슈 없으면 짧아도 된다. |
| **강조** | 상태 배지(완료·진행·지연·블로킹), 타임라인, 이슈 빨간 박스 |
| **어조** | 사실·객관. 이슈는 감추지 않고 조치 계획과 함께 제시. |
| **언제** | 주간·월간 프로젝트 보고, 스프린트 리뷰, 진행 상황 공유 |

**섹션 체크리스트:**
- [필수] 전체 상태 한 줄 + 상태 배지 (On Track / At Risk / Delayed)
- [필수] 이번 기간 완료·진행 항목
- [필수] 이슈·리스크 (없으면 "없음"으로 명시)
- [필수] 다음 기간 계획 또는 조치 사항
- [선택] 일정·마일스톤 타임라인

---

## 결 선택 휴리스틱

- 경영진·임원에게 → **임원 요약결** (길면 읽히지 않는다)
- 팀원·실무자에게 · 근거가 중요할 때 → **분석 보고결**
- 주기 보고·프로젝트 관계자에게 → **현황 보고결**
- 독자가 섞여 있으면 → 임원 요약결로 만들고, 분석 보고결을 첨부

## 결 + 밀도 조합

| 결 | 압축 | 표준 | 상세 |
|----|------|------|------|
| 임원 요약 | 결론 + 수치 2개 + 권고 | 결론 + 수치 3개 + 근거 요약 + 권고 | (비권장: 임원에게 긴 보고서는 역효과) |
| 분석 보고 | 요약 + 주요 인사이트 3개 | 전체 섹션 | 데이터 원본 + 방법론 전부 |
| 현황 보고 | 상태 배지 + 이슈 + 다음 계획 | 전체 섹션 | 마일스톤 상세 + 리스크 매트릭스 |

## 새 결을 추가하려면

결은 독자 가정·섹션 순서·강조 요소·어조·언제 쓰는지를 모두 정의한 **완성형 명세**와 **HTML 템플릿 1개**로 구성된다. 추가 시 verify.sh를 반드시 통과시킨다.
