# /report : HTML 보고서 제작

분석·현황·조사·제안을 **외부 의존성 0개의 단일 HTML 파일**로 만든다. 파일 하나만 보내면 받는 사람이 어떤 환경에서든 그대로 연다. 빌드·설치·인터넷 불필요.

> 이 스킬의 목표는 "정보를 담는 것"이 아니라 **"읽는 사람이 10초 안에 핵심을 파악하고 다음 행동을 결정하게 하는 것"**이다.

## 제1원칙 : 결론이 먼저다 (역피라미드)

서론부터 빌드업하지 않는다. **첫 화면에 "그래서 무엇"이 보여야 한다.** 어떤 결을 선택해도 이 원칙은 변하지 않는다. 독자가 결론만 읽고 의사결정해도 된다. 본문은 근거다.

## 절차 (이 순서를 지킨다)

1. **인터뷰** : 아래 [깊은 인터뷰]로 결·독자·목적을 먼저 끌어낸다.
2. **결론 확정** : 보고서를 쓰기 전에 "한 줄 결론"을 먼저 쓴다 (전체가 이 한 줄을 위해 존재한다).
3. **구조 설계** : 고른 결의 골격에 맞춰 섹션을 배치한다 → `reference/report-grains.md`
4. **제작** : 해당 결의 `templates/report.<결>.html.txt`를 복제해 채운다.
5. **검증** : `scripts/verify.sh`로 규약 위반을 막고, [육안 체크리스트]로 핵심 파악 속도를 본다.

## 깊은 인터뷰 (옵션은 고르게, 내용은 받는다)

선택지가 있는 건 **선택 창**으로 띄워 고르게 한다. 직접 받는 건 **실제 보고 내용**이다.

**0단계 : 보고서 결 + 밀도 (선택 창으로 고르게 한다)**

결은 구조와 시각 언어를 통째로 정의한다. 색만 다른 게 아니라 섹션 순서·강조 요소·밀도 기준이 다르다. 정의·토큰값은 `reference/report-grains.md`.

- **보고서 결** : 독자·목적에 맞는 것을 고른다
  - ① 임원 요약결 : 결론 먼저·핵심 수치·권고 행동 (의사결정자용, 1페이지 피팅)
  - ② 분석 보고결 : 배경→데이터→인사이트→제언 (실무자용, 근거 상세)
  - ③ 현황 보고결 : 현황→이슈→조치→일정 (프로젝트 관계자용, 주기 보고)
- **밀도** : 압축(요점만) / 표준 / 상세(근거 전부)

**1단계 : 독자·목적 (여기서 깊이가 갈린다)**

- **독자** : 경영진(결론·의사결정) / 실무자(상세·근거) / 외부·고객(맥락 포함) / 혼합
- **보고서의 목적** : 의사결정 요청 / 현황 공유 / 승인 요청 / 분석 결과 전달
- 독자가 이 보고서를 받고 **무엇을 해야 하는가** (마지막 섹션의 촉구를 정한다)

**2단계 : 핵심 내용 (필수)**

- **한 줄 결론** : 이 보고서가 전하는 단 하나의 메시지
- **핵심 수치 또는 근거** : 결론을 뒷받침하는 가장 강한 데이터 3개 이내
- **권고 행동 또는 요청 사항** : 독자가 해야 할 일

**3단계 : 상세 내용**

- 배경·맥락 (왜 이 보고가 필요한가)
- 상세 분석 또는 현황 데이터
- 리스크·한계·주의 사항
- 출처·방법론 (있을 경우)

## 디자인 시스템 (결마다 구조가 다름, 기술 규약은 공통)

3개 결은 **같은 HTML 기술 기반**을 쓰되 섹션 순서·강조 요소·시각 언어가 다르다. CSS는 `<style>` 인라인, 외부 CDN 금지.

**공통 기술 규약:**
- 외부 CDN script 0개, 외부 CSS 0개 (자체완결)
- `@media print` 인쇄 대응 필수
- 제목 `<h1>` 필수
- em-dash·en-dash 금지, 구분자는 콜론(`:`)

**컴포넌트 클래스** (3개 결 공통 어휘):

| 클래스 | 용도 |
|--------|------|
| `.summary-box` | 핵심 요약 (결론 먼저) |
| `.kpi-row` | 핵심 수치 강조 (숫자 크게) |
| `.section` | 본문 섹션 |
| `.data-table` | 데이터 표 |
| `.status-badge` | 상태 배지 (완료·진행·지연) |
| `.risk-box` | 리스크·주의 강조 |
| `.action-box` | 권고 행동·요청 |
| `.timeline` | 일정·마일스톤 |
| `.footnote` | 출처·주석 |

결별 상세 : `reference/report-grains.md`

## 안티패턴 (이게 보이면 다시 만든다)

- 첫 화면이 배경 설명으로 시작 → 결론이 먼저다. 배경은 접어두거나 뒤로 뺀다.
- 수치에 출처가 없음 → 근거 없는 수치는 설득력이 없다.
- 제목이 "OO 보고서" → "OO로 인해 Q3 매출 12% 감소"처럼 결론이 제목이다.
- 임원에게 분석 보고결 → 임원은 결론만 읽는다. 결부터 맞게 골라야 한다.
- "결론 및 제언" 섹션이 마지막 → 임원 요약결에서는 이것이 첫 섹션이다.
- 밀도 "압축"인데 5페이지 → 압축은 핵심만이다. 결론 + 수치 3개 + 권고 1개.

## 검증 (필수 : loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <만든-보고서.html>
```

자동 검사(verify.sh) + 육안 체크리스트를 **둘 다** 통과해야 완료다.

- [ ] (자동·HARD) 외부 CDN script/stylesheet 0건
- [ ] (자동·HARD) em-dash/en-dash 0건
- [ ] (자동·HARD) `<h1>` 존재
- [ ] (자동·WARN) `<style>` 인라인 존재
- [ ] (자동·WARN) `@media print` 존재
- [ ] (자동·WARN) `.summary-box` 또는 요약 섹션 존재
- [ ] (자동·WARN) `.action-box` 또는 권고 섹션 존재
- [ ] (육안) 10초 안에 결론이 보임
- [ ] (육안) 모든 수치에 출처 또는 한계 명시
- [ ] (육안) 독자가 마지막에 무엇을 해야 하는지 명확
