# /report : 단일 HTML 보고서

분석·현황·조사 결과를 **외부 의존성 0개의 단일 HTML 파일**로 만든다.
이 파일 하나만 보내면 받는 사람이 어떤 환경에서든 그대로 연다. 빌드·설치·인터넷 불필요.

## 산출물

- `.html` 파일 1개 (CSS 인라인, 외부 폰트·CDN·스크립트 불러오지 않음)
- 인쇄(PDF 저장) 친화적 레이아웃
- 제목 → 요약 → 본문 섹션 → 근거/출처 구조

## 왜 단일 파일인가

loop-kit의 "의존성 제로" 철학을 가장 잘 보여주는 산출물이다.
외부 리소스를 하나라도 참조하면 그 링크가 죽는 순간 보고서가 깨진다.
모든 것을 파일 안에 넣어 **회수·아카이브·재배포가 자유로운** 문서를 만든다.

## 구조 (권장)

```
1. 헤더    : 제목 · 작성일 · 작성자
2. 요약    : 3~5줄. 결론 먼저.
3. 본문    : 섹션별 소제목 + 근거. 표·수치는 인라인 스타일로.
4. 근거    : 데이터 출처 · 가정 · 한계
```

## 절차

1. 결론을 먼저 정한다 (요약이 본문보다 먼저 읽힌다).
2. 본문을 섹션으로 나누고 근거를 붙인다.
3. CSS를 `<style>`에 인라인으로 작성한다 (외부 폰트·CDN 금지).
4. 검증 스크립트로 자체완결성을 확인한다 (아래 [검증]).
5. 발행 전 `humanize`로 문체를 다듬는다.

## 구두점 규칙 (필수)

라벨·부연의 구분자는 **콜론(`:`)**. em-dash(`—`)·en-dash(`–`) 금지.

## 검증 (필수 — loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <만든-보고서.html>
```

검사 항목:

- [ ] (HARD) 외부 CDN script/stylesheet 0건 → 의존성 제로
- [ ] (HARD) em-dash/en-dash 0건 → 콜론 사용
- [ ] (WARN) 인라인 `<style>` 존재 → 자체완결 스타일
- [ ] (사람) 결론이 요약에 먼저 나옴
- [ ] (사람) 모든 수치에 출처·근거 명시

## 체인 예시

```
analyze  →  report  →  humanize  →  공유
research →  report  →  발행
```
