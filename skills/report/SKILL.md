# /report : 단일 HTML 보고서

분석·현황·조사·제안을 **외부 의존성 0개의 단일 HTML 파일**로 만든다. 파일 하나만 보내면 받는 사람이 어떤 환경에서든 그대로 연다. 빌드·설치·인터넷 불필요.

## 제1원칙 : 결론 먼저 (역피라미드)

서론부터 빌드업하지 않는다. **첫 화면에 "그래서 무엇"**이 보여야 한다. 바쁜 독자는 요약만 읽고 의사결정한다. 구조·종류별 골격은 `reference/structures.md`.

## 인터뷰 (옵션은 고르게, 내용은 받는다)

선택지가 있는 건 **선택 창**으로 띄워 고르게 한다(번호 나열로 서술 유도 금지). 직접 받는 건 **실제 보고 내용**이다.

- **보고서 종류** : 분석 / 현황 / 조사 / 제안  → 골격이 달라짐
- **독자** : 경영진(결론·의사결정) / 실무(상세·근거) / 외부(맥락 포함)  → 깊이가 달라짐
- **밀도** : 1페이지 요약 / 표준 / 상세
- **(데이터)** 실제 수치·발견·근거 : 직접 입력

## 산출물

- `.html` 파일 1개 (CSS 인라인, 외부 폰트·CDN·스크립트 0개)
- 인쇄(PDF 저장) 친화적 (`@media print`)
- 구조 : 제목 → 요약(결론) → 본문 → 근거·한계 → 권고
- 베이스 : `templates/report.html.txt`를 복제해 채운다

## 절차

1. 인터뷰로 종류·독자·밀도를 받는다.
2. **결론을 먼저 확정**한다 (요약이 본문보다 먼저 읽힌다).
3. 종류별 골격(`reference/structures.md`)에 맞춰 본문을 짠다.
4. 모든 수치에 **출처·한계**를 단다.
5. CSS를 `<style>`에 인라인 (외부 폰트·CDN 금지).
6. 검증 스크립트로 자체완결성을 확인하고, 발행 전 `humanize`로 문체를 다듬는다.

## 구두점 규칙

라벨·부연 구분자는 **콜론(`:`)**. em-dash(`—`)·en-dash(`–`) 금지.

## 검증 (필수 — loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <만든-보고서.html>
```

- [ ] (HARD) 외부 CDN script/stylesheet 0건 → 의존성 제로
- [ ] (HARD) em-dash/en-dash 0건 → 콜론
- [ ] (WARN) 인라인 `<style>` 존재 · 제목 `<h1>` 존재 · `@media print` 대응
- [ ] (육안) 결론이 요약에 먼저 나옴 (역피라미드)
- [ ] (육안) 모든 수치에 출처·한계 명시

## 체인 예시

```
analyze  →  report  →  humanize  →  공유
research →  report  →  발행
```
