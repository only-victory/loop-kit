## 각 칸을 어떻게 채우나

- **입력계약** : 이 단계가 받는 것의 형태. "후보 글감 N건 + 출처 URL"처럼 구체적으로. "데이터" 같은 모호어 금지.
- **처리** : 무슨 일을 하나. 변환 단계면 **어떤 스킬을 호출하는지** 명시 (humanize·report·slide). 소스·싱크는 `(어댑터)`로 표시 = 환경마다 갈아끼우는 부분.
- **출력계약** : 이 단계가 내보내는 것의 형태. 다음 단계의 입력계약과 맞아야 한다.
- **게이트** : 이 출력이 통과해야 할 기준. "출처 있음·당일치", "슬롭 패턴 0", "verify 통과"처럼 **판정 가능한** 문장으로.
- **담당** : `기계` 또는 `사람`. **사람은 판단 지점(검수·승인)에만.** 노동(서치·포맷·게시)은 기계.

## 규칙

1. **사람 체크포인트 최소 1개.** 전부 기계면 슬롭을 스케일하는 것이다. 린터가 막는다.
2. **외부 공개물은 승인(사람)이 싱크 앞에 있어야 한다.** 자동 게시 금지.
3. **소스·싱크는 어댑터.** 실제 도구(서치 엔진·게시 대상)는 루프 정의가 아니라 환경에 꽂는다. 그래야 범용이다.
4. **출력계약 = 다음 입력계약.** 단계 사이가 맞물려야 파이프라인이 끊기지 않는다.
5. 라벨 구분자는 콜론(`:`). em-dash·en-dash 금지.

## 다 채웠으면

```bash
bash ../scripts/verify.sh <루프이름>.md
```

HARD 0건이면 정의 완성. 그다음 소스·싱크 어댑터를 환경에 연결하고 1회 수동 실행으로 끝까지 돌려본 뒤, 그 흔적을 `examples/<루프이름>-run.md`로 남긴다.
