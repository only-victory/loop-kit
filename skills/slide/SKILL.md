# /slide : HTML 슬라이드 제작

원거리 시청(프로젝터·대형 모니터)에 최적화된 다크 테마 슬라이드를 **단일 HTML 파일**로 만든다. 외부 의존성 없이 브라우저로 바로 연다.

> 이 스킬의 목표는 "슬라이드를 뽑는 것"이 아니라 **청중을 한 줄의 결론으로 데려가는 것**이다.
> 예쁜 장표 50장보다, 메시지가 흐르는 12장이 낫다.

## 제1원칙 : 슬라이드 1장 = 메시지 1개

한 장에 주장이 둘이면 청중은 둘 다 놓친다. 장표를 만들기 전에 **"이 장이 전하려는 한 문장"**을 먼저 쓰고, 그 문장이 제목이 되게 한다. 그 문장으로 요약이 안 되면 장을 쪼갠다.

## 절차 (이 순서를 지킨다)

1. **인터뷰** : 아래 [깊은 인터뷰]로 목적·청중·행동을 먼저 끌어낸다. 주제만 듣고 만들지 않는다.
2. **내러티브 설계** : 장 단위로 뽑기 전에 *이야기 골격*부터 짠다 → `reference/narrative.md`
3. **레이아웃 매핑** : 각 메시지에 맞는 레이아웃을 고른다 (내용이 레이아웃을 정한다) → `reference/layouts.md`
4. **제작** : `templates/deck.html.txt`를 베이스로 디자인 토큰에 맞춰 채운다.
5. **검증** : `scripts/verify.sh`로 규약 위반을 막고, [육안 체크리스트]로 메시지·가독성을 본다.

## 깊은 인터뷰 (주제만 듣고 시작하지 않는다)

슬라이드가 평범해지는 가장 큰 이유는 **요구사항을 얕게 받아서**다. 아래를 단계로 묻는다.
이미 사용자가 답한 항목은 건너뛰고, 빈 항목만 묻는다.

> **선택지가 정해진 질문(색·톤·밀도·발표 목적·청중 수준)은 반드시 선택 창(구조화 선택지 도구)으로 띄워** 고르게 한다. 번호 목록 텍스트로 나열해 사용자가 서술하게 만들지 말 것. 사용자가 직접 입력해야 하는 건 **실제 주제·핵심 메시지·본문 내용뿐**이다. (선택 UI를 못 쓰는 환경에서만 번호 목록으로 폴백)

**0단계 : 스타일 (선택 창으로 고르게 한다 — 자유 입력 금지)**

자유 입력을 받으면 색·톤이 빈약해진다. 아래를 **선택 창으로 띄워** 고르게 한다. 프리셋 정의·토큰값은 `reference/themes.md`.

- **색 테마** : ① 딥 다크·블루퍼플(기본) ② 미드나잇 모노 ③ 웜 앰버 ④ 하이 콘트라스트 ⑤ 라이트
- **톤** : 임팩트 / 교육 / 보고 / 스토리
- **밀도** : 압축(5~8장) / 표준(10~15장) / 상세(20장+)

고른 색 테마의 `:root` 블록을 `templates/deck.html.txt`에 통째로 적용하고, 톤·밀도로 레이아웃·텍스트량·장 수를 정한다.

**1단계 : 뼈대 (필수)**
- 주제와 **핵심 메시지 한 문장** (장표 수가 아니라 이 한 문장이 덱을 지배한다)
- 분량 (대략 장 수)
- 대상 (누가 보나)

**2단계 : 목적·청중 (여기서 깊이가 갈린다)**
- 발표 **목적** : 설득 / 교육 / 보고 / 공유 중 무엇? (목적이 내러티브 골격을 바꾼다)
- 청중의 **사전 지식** : 입문 / 중급 / 전문가? (용어·전제의 깊이를 정한다)
- 발표 후 청중이 **무엇을 하길** 바라나? (마지막 장의 행동 촉구가 여기서 나온다)

**3단계 : 톤·제약 (선택)**
- 발표 시간 (장당 1~2분 기준 → 적정 장 수 역산)
- 반드시 강조할 포인트 / 절대 빼면 안 되는 것
- 피해야 할 것 (민감 정보, 특정 표현)

> 2단계를 건너뛰면 "정보 나열 덱"이 나온다. 목적·청중·행동 세 가지는 꼭 채운다.

## 디자인 토큰 (요약 — 상세는 reference)

```css
--bg-base:#0a0a0f; --text-hi:#f5f5fa; --text-md:#cfcfdc; --text-lo:#8a8aa0;
--accent-grad:linear-gradient(90deg,#5b8cff,#a371ff);
--success:#4ade80; --warning:#fbbf24; --danger:#ff6b7a;
```

- 타이포 : Hero `clamp(58px,8vw,114px)` / H1 60px / 본문 18–20px. **이보다 작게 쓰지 않는다.**
- 간격 : `4/8/12/16/24/32/48/64/96/128px`만.
- 구두점 : 라벨 구분자는 **콜론(`:`)**. em-dash(`—`)·en-dash(`–`) 금지.
- 모티프 : 좌상단 `✦`, 우하단 카운터 `{현재} / {전체}`.

위 값은 **기본 테마(①)**다. 0단계에서 고른 색 테마로 `:root`를 교체한다 (5종 토큰: `reference/themes.md`).
전체 토큰·레이아웃별 토큰은 `templates/deck.html.txt`에 적용되어 있다.

**★ 레이아웃 일관성 (필수)** : 모든 슬라이드는 템플릿의 **고정 프레임**을 쓴다 — `.frame` 안에 `.eyebrow` → `.title` → `.lead`/`.body` 순서. 좌측 거터(`--gutter-x`)와 제목 높이(`--anchor-y`)가 **장마다 동일**해 넘겨도 제목이 흔들리지 않는다. **슬라이드별로 padding·위치·정렬을 새로 잡지 말 것** (세로 중앙 정렬 금지 — 내용 높이가 달라 제목이 들썩인다).

## 안티패턴 (이게 보이면 다시 만든다)

- 한 장에 글머리 기호 7개 폭탄 → 3개 이하, 아니면 쪼갠다
- 제목이 명사구("매출 현황") → 메시지 문장("매출은 3분기에 꺾였다")으로
- 본문에 문단 통째 → 슬라이드는 말의 보조물, 발표자가 읽는 대본이 아니다
- 14px 미만 본문 → 뒷자리에서 안 읽힌다
- 레이아웃 1종으로 끝까지 → 내용이 바뀌면 레이아웃도 바뀐다

## 검증 (필수 — loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <만든-슬라이드.html>
```

자동 검사(verify.sh) + 육안 체크리스트를 **둘 다** 통과해야 완료다.

- [ ] (자동·HARD) em-dash/en-dash 0건, 외부 CDN 0건, 슬라이드 1장 이상
- [ ] (자동·WARN) `✦` 모티프, 키보드 내비, `prefers-reduced-motion`, 14px 미만 본문
- [ ] (육안) 모든 장이 "한 문장"으로 요약됨 (1장 1메시지)
- [ ] (육안) 마지막 장에 청중의 다음 행동이 있음
- [ ] (육안) 프로젝터 뒷자리에서 제목·본문이 읽힘
