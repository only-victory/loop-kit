# 디자인 결 : 색이 아니라 인상을 고른다

인터뷰 0단계에서 아래 **디자인 결**을 선택 창으로 제시하고, 고른 결의 템플릿을 복제한다. 자유 입력이 아니라 **고르게** 한다.

기존엔 색 5종을 `:root`만 바꿔 적용했는데, 그건 골격이 같아 어떤 색을 골라도 같은 인상이었다. 그래서 색을 **결 안으로 흡수**했다. 결은 색만이 아니라 **폰트·타이포 스케일·그리드 거동·도형 언어·모티프**까지 통째로 정의한다. 결이 다르면 같은 마크업도 전혀 다른 덱이 된다.

각 결은 같은 클래스 어휘를 공유하므로 내용 작성법은 동일하다. 바뀌는 건 `<style>`과 폰트 link 뿐이다.

## 결 4종

### ① 에디토리얼 (다크 지면)
`templates/deck.editorial.html.txt`

| 축 | 값 |
|----|----|
| 폰트 | 디스플레이 Fraunces(세리프, 이탤릭 강조) + 본문 Pretendard |
| 색 | 따뜻한 잉크 배경 `#12110c`, 종이빛 글자, 금 `#d6a64f`·세이지 `#86a594` 강조 |
| 그리드 | 비대칭, 본문 measure 60ch, 헤어라인 rule |
| 도형 | 최소 라운드, 1px 라인, 큰 여백 |
| 모티프 | 금색 `✦`, 우상단 폴리오, eyebrow 뒤 헤어라인 |
| 언제 | 칼럼·에세이·브랜드 발표, 차분하고 격있는 인상 |

## 톤 (4종) : 결 위에 얹는 생성 편향

| 톤 | 레이아웃 편향 | 텍스트 |
|----|---------------|--------|
| **임팩트** | hero·stats-split 多, 장당 1메시지 엄격 | 최소 (키워드 중심) |
| **교육** | content·steps-list·예시 多 | 친절한 설명문, 용어 풀이 |
| **보고** | comparison-table·수치, 결론 먼저 | 밀도 높음, 근거·출처 명시 |
| **스토리** | quote·전환 강조, 훅 강하게 | 내러티브 흐름 |

## 밀도 (3종) : 장 수

| 밀도 | 장 수 | 용도 |
|------|-------|------|
| **압축** | 5~8장 | 핵심만, 짧은 발표 |
| **표준** | 10~15장 | 일반 발표 |
| **상세** | 20장+ | 교육·워크숍 |

## 적용 순서

1. 디자인 결 1개 → 해당 `templates/deck.<결>.html.txt` 복제 (색·폰트 포함)
2. 톤 1개 → 레이아웃·텍스트량 편향 반영
3. 밀도 1개 → 장 수 결정
4. 그다음 내용 인터뷰(주제·목적·청중)로 진행

## 새 결을 추가하려면

결은 `:root` 토큰 + 컴포넌트 스타일 + 폰트 link만 다른 **완성형 템플릿 1개**다. 추가 시 `verify.sh`를 반드시 통과시킨다 (em/en-dash 0건, 허용 CDN만, `✦`·키보드 내비·`prefers-reduced-motion`, 14px 미만 본문 0건). 검증 안 된 결은 추가하지 않는다.

## 확장 컴포넌트 카탈로그

4개 결은 가장 자주 쓰는 컴포넌트를 공통으로 담는다. 더 특수한 것(diagram-flow, definition-card, prompt-box, failure-mode, session-index, curriculum, assignment, image, blank)은 레거시 카탈로그 `templates/deck.html.txt`(edu 네온 팔레트)에 정의돼 있으니, 필요할 때 해당 마크업·CSS를 골라 현재 결의 토큰에 맞춰 가져온다.
