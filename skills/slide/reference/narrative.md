# 내러티브 설계 : 장표를 뽑기 전에 이야기부터

슬라이드는 장표의 모음이 아니라 **하나의 논증**이다. 청중은 슬라이드를 "읽는" 게 아니라 발표자의 흐름을 "따라온다". 그래서 장을 만들기 전에 흐름을 먼저 설계한다.

## 기본 골격 (목적에 따라 변형)

```
훅      : 왜 지금 이 이야기를 듣나 (한 장, 강한 한 문장 or 수치)
맥락    : 무엇이 문제이고 왜 중요한가
핵심 주장: 이 발표가 전하는 단 하나의 결론 (가장 중요한 장)
근거    : 주장을 떠받치는 3개 (3개를 넘기면 청중이 잊는다)
함의    : 그래서 무엇이 달라지나
행동    : 청중이 지금 할 일 (마지막 장)
```

## 목적별 골격 변형

| 목적 | 강조 | 골격 조정 |
|------|------|-----------|
| **설득** | 주장 → 근거 → 행동 | 반론을 먼저 인정하고 깨는 장을 근거 앞에 넣음 |
| **교육** | 개념 → 예시 → 실습 | 근거 자리에 "예시"와 "직접 해보기"를 배치 |
| **보고** | 결론 → 근거 → 리스크 | 훅 대신 결론을 1장에 먼저 (바쁜 청중) |
| **공유** | 맥락 → 발견 → 다음 | 행동 촉구를 약하게, "함께 보면 좋을 것"으로 |

## 장 수 역산

발표 시간 ÷ (장당 1~2분) = 적정 장 수. 시간이 모자라면 **장을 줄이지 말고 근거를 줄인다**(3개 → 2개). 흐름이 끊기는 것보다 근거가 적은 게 낫다.

## 흐름 점검 (만든 뒤)

- 각 장 제목만 위에서 아래로 읽었을 때 **하나의 이야기로 읽히는가?**
- 빼도 흐름이 안 무너지는 장이 있으면 → 뺀다
- "핵심 주장" 장을 가리켰을 때 발표 전체가 이 장을 위한 것이었나?
