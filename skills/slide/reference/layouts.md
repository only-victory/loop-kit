# 레이아웃 선택 : 내용이 레이아웃을 정한다

레이아웃을 먼저 고르고 내용을 끼워넣지 않는다. **전하려는 메시지의 형태**가 레이아웃을 정한다. 아래는 "이 메시지엔 이 레이아웃"의 매핑이다.

| 레이아웃 | 언제 쓰나 | 한 줄 예 | 안티 |
|----------|-----------|----------|------|
| `hero` | 섹션을 여는 임팩트, 단 하나의 선언 | "AI는 도구가 아니라 동료다" | 본문 채우기 |
| `title` | 단순 구획 전환 | "2부 : 실전 적용" | 정보 담기 |
| `content` | 메시지 + 근거 2~3개 | 주장 + 불릿 3개 | 불릿 7개 |
| `two-column` | 대비·병렬 (전/후, A/B) | "기존 vs 개선" | 무관한 두 내용 |
| `stats-split` | 수치 하나로 충격 | "응답 78%" 크게 | 표로 될 걸 키운 것 |
| `comparison-table` | 2~3개 항목 다축 비교 | 플랜 3종 비교표 | 5개 이상(복잡) |
| `steps-list` | 순차 절차 | "1 → 2 → 3" | 순서 없는 목록 |
| `principle` | 원칙의 Good/Bad 대비 | "이렇게(O) / 저렇게(X)" | 중립 설명 |
| `definition-card` | 용어 한 개 정의 | "MCP란 :" | 여러 용어 나열 |
| `diagram-flow` | 관계·흐름·구조 | 노드-엣지 다이어그램 | 텍스트로 될 것 |
| `quote` | 권위 인용·전환점 | 고객/전문가 한마디 | 출처 불명 |
| `code` / `code-split` | 코드, 또는 코드+결과 | 예제 + 출력 나란히 | 긴 코드 전체 |
| `module-intro` | 모듈 시작 (글로우 배경) | "Module 3 : 보안" | 본문 슬라이드 |
| `session-index` | 세션 번호 + 대제목 전면 (좌측 정렬) | "SESSION 01 : 기초" | 본문 슬라이드 |
| `curriculum` | 커리큘럼 목차 (번호 + 제목 + 부연) | 강의 전체 목차 | 내용 없는 장식 |
| `image` | 이미지 중심 + 캡션 | 스크린샷, 다이어그램 사진 | 텍스트가 더 많을 때 |
| `blank` | 완전 자유 레이아웃 (`.frame` 없음) | 직접 배치가 필요한 복잡한 구성 | 기본 레이아웃으로 될 것 |
| `failure-mode` | 실패 패턴 강조 (빨간 좌측 보더) | "이 방식은 반드시 실패한다" | Good/Bad 대비가 필요할 때 (`principle` 사용) |
| `prompt-box` | 프롬프트 예시 박스 (모노스페이스) | 실제 입력 예시, 템플릿 | 일반 코드 블록(`code`) |
| `assignment` | 과제 체크리스트 | 워크숍 실습, 액션 아이템 | 단순 불릿 목록 |
| `outro` | 마무리 + 행동 촉구 | "오늘 할 일 : ..." | 단순 "감사합니다" |

## 선택 휴리스틱

- 숫자가 주인공이면 → `stats-split` (표 아님)
- 둘을 견주면 → 항목 2개는 `two-column`, 3개+다축은 `comparison-table`
- 순서가 의미를 가지면 → `steps-list`
- "이렇게 하라/하지 마라"면 → `principle`
- 관계가 핵심이면 → `diagram-flow` (말로 풀지 말 것)
- 한 문장으로 압도하려면 → `hero`

## 다양성 규칙

같은 레이아웃이 4장 연속이면 청중이 지친다. 흐름상 자연스럽다면 중간에 `hero`·`quote`·`stats-split`로 **호흡을 바꾼다**. 단, 다양성을 위해 억지 레이아웃을 쓰지는 않는다 — 내용이 먼저다.
