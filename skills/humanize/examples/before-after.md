# humanize 예시 : before → after (등급 D → A)

## Before (등급 D)

> 이번 글에서는 원격 근무에 대해 알아보겠습니다. 원격 근무의 중요성은 아무리
> 강조해도 지나치지 않습니다. 혁신적인 제도로서 다음과 같습니다: 생산성이
> 향상된다고 할 수 있습니다.

```
심각도 : S1 3 · S2 2 · S3 0    등급 : D
✗ S1(치명) 잔존 → humanize 미통과
```
S1: 대해 알아보 / 알아보겠습니다 / 아무리 강조해도 · S2: 혁신적 / 다음과 같습니다:

## After (등급 A)

> 원격 근무는 두 가지를 바꿨다. 출퇴근 두 시간이 일하는 시간으로 돌아왔고,
> 임대료가 사라졌다. 다만 규칙 없이 자율에만 맡기면 협업이 끊긴다.

```
심각도 : S1 0 · S2 0 · S3 0    등급 : A
변경률 19% (원문 100자 → 81자)
✓ humanize 통과 (등급 A)
```

## 무엇이 달라졌나

- 의례적 도입("~에 대해 알아보겠습니다") 삭제, 본론을 첫 문장에
- 과장 상투어·빈 수식어("혁신적인")·헤지("~라고 할 수 있습니다") 제거
- 추상 나열 → 구체("두 시간"·"임대료")
- 장점만 늘어놓던 글에 한계("협업이 끊긴다")를 더해 사람의 판단을 드러냄
- 변경률 19% : 의미를 보존하면서 다듬음 (과잉 재작성 아님)
