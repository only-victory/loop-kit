# /humanize : AI 슬롭 제거

## 언제 쓰나

텍스트 산출물이 완성된 뒤, 공개·발행 직전의 마지막 단계. 선택이 아니라 기본 공정이다.
`blog`·`report`·`email` 등 다른 스킬의 출력을 받아 마지막에 통과시킨다.

## 무엇이 달라지나

- "살펴보겠습니다" 류 의례적 도입 : 삭제하거나 본론을 바로 서술
- 수동태 남용 : 능동형으로 전환
- 목록 남발 : 산문으로 통합 (정말 병렬일 때만 목록 유지)
- 빈 수식어 ("혁신적인", "체계적인", "종합적인") : 구체로 대체하거나 제거
- 불필요한 영어 차용어 ("레버리지", "인사이트") : 우리말로
- 헤지 표현 ("~라고 할 수 있습니다", "것으로 보입니다") : 단정하거나 근거 제시

## 절차

1. 원문을 읽고 아래 [금지 패턴]을 진단한다 (어디서 몇 번 나오는지).
2. 의미를 바꾸지 않으면서 패턴을 제거·교체한다.
3. 소리 내어 읽었을 때 자연스럽게 흐르는지 점검한다.
4. 검증 스크립트로 금지 패턴이 0건인지 확인한다 (아래 [검증]).

## 금지 패턴 (즉시 제거)

```
의례적 도입 : ~을 살펴보겠습니다 / ~에 대해 알아보겠습니다
과장 상투어 : 중요성은 아무리 강조해도 지나치지 않습니다 / 결론적으로
기계적 연결 : 다음과 같습니다: / 종합적으로
빈 수식어   : 혁신적인 · 획기적인 · 체계적인 · 종합적인
헤지        : ~라고 할 수 있습니다 / ~것으로 보입니다
영어 차용   : 레버리지 · 인사이트
구두점      : em-dash(—) · en-dash(–)  →  콜론(:)
```

## 출력 형식

1. **진단 요약** : 발견한 패턴 + 심각도(낮음·보통·높음)
2. **수정 전문** : 다듬은 텍스트 전체
3. **주요 변경 3~5개** : 한 줄씩

## 검증 (필수 — loop-kit 게이트)

수정한 텍스트를 파일로 저장한 뒤 번들 스크립트로 검증한다. 통과 = exit 0.

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <수정한-파일-경로>
```

검증 기준 체크리스트:

- [ ] `verify.sh`가 exit 0 (금지 패턴 0건)
- [ ] 수정 전·후 의미가 달라진 부분 없음
- [ ] 원문 길이의 ±20% 이내
- [ ] 소리 내어 읽었을 때 자연스럽게 흐름

> 스크립트가 잡지 못하는 것(의미 보존·자연스러움)은 사람·모델이 위 체크리스트로 확인한다.
> 스크립트는 "기계적으로 확실히 틀린 것"만 막는다. 둘을 합쳐야 검증이 완성된다.

## 체인 예시

```
blog   →  humanize  →  발행
report →  humanize  →  공유
email  →  humanize  →  전송
```
