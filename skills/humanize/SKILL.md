# /humanize : 문체 재설계

슬롭은 "틀린 글"이 아니라 **사람이 안 쓰는 글**이다. 그런데 슬롭을 지우는 것만으로는 부족하다 : 지운 자리에 **그 글이 살아야 할 목소리**가 들어와야 한다. 이 스킬은 "슬롭 제거"에서 **"문체 결(文體 結) 설계"**로 격이 다르다.

> 이 스킬의 목표는 어색함을 없애는 것이 아니라, **읽는 사람이 쓴 사람을 느끼게 하는 것**이다.

## 제1원칙 : 문체 결이 먼저다

글을 고치기 전에 "어떤 목소리여야 하는가"를 확정한다. 결이 정해지면 어휘·리듬·단락 구조가 따라온다. 결 없이 슬롭만 지우면 무색무취의 글이 된다.

## 절차 (이 순서를 지킨다)

1. **인터뷰** : 아래 [깊은 인터뷰]로 결·목적·독자를 먼저 끌어낸다.
2. **진단** : 원문의 슬롭을 심각도(S1/S2/S3)별로 찾는다 → `reference/patterns.md`
3. **재설계** : 고른 결의 어휘·리듬·금지 사항으로 다시 쓴다 → `reference/writing-grains.md`
4. **소리 검증** : 소리 내어 읽어 결의 리듬이 흐르는지 확인한다.
5. **게이트** : `scripts/verify.sh`로 S1 잔존 여부·변경률을 확인한다.

## 깊은 인터뷰 (옵션은 고르게, 텍스트는 받는다)

선택지가 있는 건 **선택 창**으로 띄워 고르게 한다. 직접 입력은 **원문뿐**이다.

**0단계 : 문체 결 (선택 창 필수)**

결은 색이 아니라 **목소리 전체**다. 어휘·문장 길이·단락 구조·금지 어법이 결마다 통째로 다르다. 정의·예시는 `reference/writing-grains.md`.

- ① **공식 문서결** : 보고서·기획서·공문 (격식 한자어·완결 어미·수동 허용)
- ② **에세이·칼럼결** : 블로그·뉴스레터·칼럼 (능동·장단 리듬·인상적 단어)
- ③ **대화·협업결** : 이메일·슬랙·팀 공유 (직접 호칭·구어체·짧고 단호)
- ④ **SNS·카드결** : 인스타·링크드인·카드뉴스 (첫 줄이 전부·한 줄 한 생각)

**1단계 : 목적·독자 (여기서 깊이가 갈린다)**

- 글의 **목적** : 설득 / 보고 / 공유·정보 / 감성 유대 (목적이 강도를 정한다)
- **독자** : 경영진 / 실무·팀원 / 외부·고객 / 불특정 다수 (독자가 어휘 수준을 정한다)
- 발행 후 독자가 **무엇을 하길** 바라나 (마지막 문장의 어조를 정한다)

**2단계 : 강도·제약**

- **강도** : 가볍게(어색한 것만) / 표준 / 강하게(목소리 전면 재설계)
- 반드시 유지할 표현 / 절대 쓰지 말아야 할 표현

> 강도는 **변경률 가드**와 연결된다 : "가볍게"인데 원문이 절반 넘게 바뀌면 과한 것이다.

**(데이터)** 다듬을 원문 : 직접 입력

## 디자인 시스템 (결마다 목소리가 다름, 진단 어휘는 공통)

4개 결은 **같은 슬롭 진단 체계**를 쓰되 재설계 방향이 다르다. 진단은 결과 무관하게 동일하고, 결을 바꾸면 같은 원문도 전혀 다른 목소리가 된다.

| 축 | 결마다 다른 것 |
|----|----------------|
| 어휘 | 한자어 비율·전문 용어·구어 허용 여부 |
| 문장 길이 | 단문 지향 / 장단 혼합 / 중·장문 |
| 단락 구조 | 두괄식 / 기승전결 / 한 문장 단락 |
| 금지 어법 | 결마다 다른 금지 목록 |
| 리듬 목표 | 낭독했을 때 달성해야 할 운율 |

전체 결 명세 : `reference/writing-grains.md`

## 출력 형식

1. **진단** : 결 적합성 평가 + 슬롭 심각도별 발견(S1/S2/S3) + 현재 등급
2. **재설계 전문** : 고른 결로 다시 쓴 텍스트 전체
3. **주요 변경 3~5개** : 무엇을 왜 바꿨나 (결 기준으로 설명)
4. **최종 등급** : A 목표, 결 달성 여부 명시

## 안티패턴 (이게 보이면 다시 만든다)

- 결 선택 없이 "그냥 자연스럽게" 수정 → 무색무취의 글이 나온다. 결부터 정한다.
- 슬롭만 지우고 결 목소리를 안 입힘 → 병원 처방전처럼 정확하지만 차갑다.
- 공식 문서결인데 구어체 동사 → 결 일관성 깨짐.
- SNS결인데 세 줄짜리 단락 → 스크롤 중 이탈한다.
- 강도 "가볍게"인데 원문의 60%를 바꿈 → 원문 의도를 벗어난 것이다.
- 독자 없이 쓴 글 → 누구를 위한 목소리인지 알 수 없다.

## 검증 (필수 : loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <다듬은-파일> [원문-파일]
```

자동 검사 + 육안 체크리스트를 **둘 다** 통과해야 완료다.

- [ ] (자동·HARD) S1 잔존 0건 : 통과
- [ ] (자동·WARN) 등급 A 목표 (B : S3만 남음, C : S2 남음, D : S1 잔존)
- [ ] (자동·WARN) 변경률이 강도에 맞음 (가볍게 + 50%↑ = 과함)
- [ ] (육안) 고른 결의 목소리가 흐름
- [ ] (육안) 소리 내어 읽어 자연스러움
- [ ] (육안) 수정 전후 의미 동일
- [ ] (육안) 독자가 마지막 줄에서 원하는 행동을 하고 싶어짐

## 체인 예시

```
blog   →  humanize(에세이결)  →  발행
report →  humanize(공식결)    →  공유
email  →  humanize(대화결)    →  전송
promo  →  humanize(SNS결)     →  링크드인 발행
slide  →  humanize(에세이결)  →  발표자 노트
```
