# /humanize : AI 슬롭 제거

슬롭은 "틀린 글"이 아니라 **사람이 안 쓰는 글**이다. 의례적 도입, 빈 수식어, 자신 없는 헤지, 기계적 리듬을 걷어내 **사람의 목소리**로 되돌린다. 모든 콘텐츠 워크플로우의 마지막 게이트.

## 인터뷰 (옵션은 고르게, 텍스트는 받는다)

선택지가 있는 건 **선택 창**으로 띄워 고르게 한다(번호 나열로 서술 유도 금지). 사용자가 직접 줘야 하는 건 **다듬을 텍스트(원문)뿐**이다.

- **문서 종류** : 블로그 / 이메일 / 보고서 / 공지·안내 / SNS  → 허용 어조의 폭을 정함
- **목표 어조** : 전문·정중 / 간결·직설 / 친근·대화체
- **강도** : 가볍게(어색한 것만) / 표준 / 강하게(재작성 수준)  → 변경 허용 범위
- **(데이터)** 다듬을 원문 : 직접 입력

> 강도는 **변경률 가드**와 연결된다. "가볍게"인데 원문이 절반 넘게 바뀌면 과한 것이다(아래 검증).

## 무엇이 달라지나 (카테고리)

의례적 도입 삭제 · 과장 상투어 제거 · 빈 수식어 → 구체 · 헤지 → 단정/근거 · 명사화·수동태 → 능동 · 목록 남발 → 산문 · 단조한 리듬 → 장단 섞기.
패턴별 before→after 전체는 `reference/patterns.md`.

## 절차

1. 인터뷰로 어조·강도를 받는다.
2. 원문을 읽고 `reference/patterns.md`로 슬롭을 **심각도(S1/S2/S3)** 별로 진단한다.
3. **의미를 보존하며** 고친다 (S1은 무조건 제거, S2·S3는 어조에 맞춰).
4. 소리 내어 읽어 리듬을 점검한다.
5. 검증 스크립트로 **등급**과 **변경률**을 확인한다.

## 출력 형식

1. **진단** : 심각도별 발견(S1/S2/S3) + 현재 등급
2. **수정 전문** : 다듬은 텍스트 전체
3. **주요 변경 3~5개** : 한 줄씩 (무엇을 왜 바꿨나)
4. **최종 등급** : A 목표

## 검증 (필수 — loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <다듬은-파일> [원문-파일]
```

- **등급** : A(슬롭 0) · B(S3만) · C(S2 남음) · D(S1 잔존 = 실패)
- **변경률** : 원문을 함께 주면 길이 변화 측정. 30%·50% 초과 시 경고(의미 보존·과잉 재작성 점검).
- 통과 = **S1 0건**(exit 0). 목표는 항상 **A**.
- 체크리스트:
  - [ ] (자동) S1 0건 → 통과, 가능하면 등급 A
  - [ ] (자동) 변경률이 강도에 맞음 (가볍게인데 50%↑면 과함)
  - [ ] (육안) 수정 전후 의미 동일
  - [ ] (육안) 소리 내어 읽어 자연스럽게 흐름

> 스크립트는 "기계적으로 확실한 슬롭"만 잡는다. 의미 보존·리듬·목소리는 사람이 본다. 둘을 합쳐야 검증이 완성된다.
