# 문체 결 : 목소리를 고른다

인터뷰 0단계에서 아래 **문체 결**을 선택 창으로 제시하고, 고른 결의 명세에 맞춰 재설계한다.

기존엔 "어조 3종"을 고르는 방식이었는데, 그건 어조만 달랐고 어휘·리듬·구조는 같았다. 그래서 어조를 **결 안으로 흡수**했다. 결은 어조만이 아니라 **어휘·문장 길이·단락 구조·금지 어법·리듬 목표**까지 통째로 정의한다. 결이 다르면 같은 원문도 전혀 다른 목소리가 된다.

---

## 결 4종

### ① 공식 문서결

**언제** : 보고서·기획서·제안서·공문·품의서·회의록

| 축 | 명세 |
|----|------|
| **어휘** | 격식 한자어 허용 (추진·검토·시행·도입), 전문 용어 풀이 없이 사용, 약어는 첫 등장 시 병기 |
| **문장 길이** | 중·장문 위주 (2~3절 복문 허용), 단문은 강조 목적으로만 |
| **어미** | 명사형 어미("…하였음", "…됨", "…할 예정"), 서술형("…합니다") 공존 허용 |
| **단락 구조** | 두괄식 필수, 단락당 논점 1개, 항목 번호 체계 적극 사용 |
| **수동·피동** | 허용 (기관·조직 주어 문장) |
| **금지** | 구어체 동사(이에요/거든요), 줄임말, 이모지, 감탄사, 의문문 단락 시작 |
| **리듬** | 낭독 시 일정하고 무게감 있는 호흡. 경쾌함 없음. |

**모델 문장 (before → after)**
- `AI를 활용한 다양한 방면의 업무 혁신이 기대됩니다` → `AI 도입으로 업무 처리 속도가 40% 향상될 것으로 예측됨`
- `진행 상황을 공유드리고자 합니다` → `3분기 진행 현황을 아래와 같이 보고함`

### ② 에세이·칼럼결

**언제** : 블로그·뉴스레터·칼럼·사내 기고·브런치

| 축 | 명세 |
|----|------|
| **어휘** | 쉬운 일상어 기반, 단락당 인상적인 단어 1개, 한자어는 꼭 필요할 때만 |
| **문장 길이** | 장단 혼합 필수 : 짧은 문장(7단어 이하) + 긴 문장(15단어+)을 교차 |
| **어미** | 능동형 단정 어미("…다", "…이다"), 의문문으로 단락 마무리 가능 |
| **단락 구조** | 기승전결 가능, 마지막 문장이 임팩트·여운, 1인칭 허용 |
| **수동·피동** | 최소화. 능동으로 바꿀 수 있으면 반드시 바꾼다 |
| **금지** | 보고서식 어미(…하였음), 형식적 도입("안녕하세요"), 과도한 헤지(것 같다), 목록 3개 이상 연속 |
| **리듬** | 낭독 시 밀고 당기는 호흡. 단조로운 리듬은 실패. |

**모델 문장 (before → after)**
- `AI는 다양한 분야에서 활발히 활용되고 있습니다` → `AI는 이제 도구가 아니다. 동료에 가깝다.`
- `본 글에서는 세 가지 방법을 소개하고자 합니다` → `방법은 셋이다. 세 번째가 핵심이다.`

### ③ 대화·협업결

**언제** : 이메일·슬랙·팀 메시지·카카오워크·내부 공지

| 축 | 명세 |
|----|------|
| **어휘** | 친근하되 명확, 줄임말 절제, 팀 전용 약어는 설명 없이 사용 가능 |
| **문장 길이** | 짧고 단호. 한 문장 = 한 요점. 복문은 두 문장으로 나눈다. |
| **어미** | 구어체 허용(…요, …죠), 직접 호칭("OO님, OO팀"), 청유문 적극 사용 |
| **단락 구조** | 핵심부터, 부연은 다음 줄, 필요하면 불릿 1단계만 |
| **수동·피동** | 금지. "전달드립니다" → "전달합니다", "공유드리고자 합니다" → "공유합니다" |
| **금지** | 딱딱한 공문체, 과도한 경어 중복("전달드리도록 하겠습니다"), 의례적 인사말 3줄 이상 |
| **리듬** | 빠르고 명확. 수신자가 스캔해도 핵심이 보여야 한다. |

**모델 문장 (before → after)**
- `항상 수고가 많으십니다. 다름이 아니라 …` → `[확인 요청] OO 건 관련 한 가지만 여쭤봐도 될까요?`
- `이와 관련하여 검토 부탁드리고자 합니다` → `내일까지 검토 부탁드려요. 의견 주시면 바로 반영할게요.`

### ④ SNS·카드결

**언제** : 링크드인·인스타그램·카드뉴스·트위터·포스트

| 축 | 명세 |
|----|------|
| **어휘** | 임팩트 단어 우선, 숫자 강조, 구체 명사 (추상 명사 최소화) |
| **문장 길이** | 한 문장이 한 줄. 최대 2줄. 3줄 넘으면 쪼갠다. |
| **어미** | 단정형("…이다", "…한다"), 의문형으로 훅("…모르고 있지는 않은가?") |
| **단락 구조** | 첫 줄 = 훅 (읽을 이유를 만든다). 각 단락 = 한 생각. 빈 줄로 호흡. |
| **수동·피동** | 전면 금지. 모든 문장 능동 주어로. |
| **금지** | 긴 서론, 중문 복문, 접속사 남발(그리고/하지만/따라서 연속), 해시태그(별도 원칙 없으면) |
| **리듬** | 스크롤하다 멈추는 리듬. 첫 줄을 놓치면 그 글은 읽히지 않는다. |

**모델 문장 (before → after)**
- `AI 기술이 발전함에 따라 우리의 업무 방식도 변화가 필요합니다` → `3년 후, 지금 하는 일의 40%는 AI가 한다.`
- `이번 세미나는 실무에 바로 적용 가능한 내용으로 구성되어 있습니다` → `배우는 날 퇴근하며 써먹는 세미나.`

## 결 선택 휴리스틱

- 공문·결재 라인이 있으면 → **공식 문서결**
- 블로그·뉴스레터로 발행하면 → **에세이·칼럼결**
- 팀원에게 보내는 이메일·슬랙이면 → **대화·협업결**
- 피드·카드뉴스로 올리면 → **SNS·카드결**
- 보고서 본문 → 공식, 보고서 요약 인트로 → 에세이, 배포용 카드 → SNS

## 결 혼합 (고급)

같은 문서에 결을 섞을 수 있다. 예 : 보고서 요약(에세이결) + 본문(공식결) + 공유 메시지(SNS결). 단, 섹션이 달라질 때마다 결을 명확히 전환한다. 한 문단 안에서 섞으면 목소리가 흔들린다.

## 새 결을 추가하려면

결은 어휘·문장 길이·어미·단락 구조·수동 기준·금지·리듬 목표를 모두 정의한 **완성형 명세**다. 추가 시 모델 문장 before→after를 최소 2쌍 포함한다. 검증 안 된 결은 추가하지 않는다.
