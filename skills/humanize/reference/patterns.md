# 한국어 AI 슬롭 패턴 카탈로그

슬롭은 "틀린 글"이 아니라 **사람이 안 쓰는 글**이다. 아래는 카테고리별 패턴 + 심각도 + 고치는 법.
심각도 : **S1**(치명, 즉시 제거 = 게이트 실패) · **S2**(강함, 권고) · **S3**(약함, 권고).

---

## 1. 의례적 도입 — S1

말을 본론 대신 의식으로 시작하는 버릇.

| 슬롭 | 사람 |
|------|------|
| ~에 대해 살펴보겠습니다 | (삭제) 본론을 첫 문장에 |
| ~에 대해 알아보겠습니다 | 바로 주장으로 |
| 지금부터 ~을 설명드리겠습니다 | 설명을 그냥 시작 |

> 도입부를 지우고 **결론·핵심을 첫 문장**에 놓으면 대부분 해결된다.

## 2. 과장 상투어 — S1

내용 없이 무게만 잡는 표현.

| 슬롭 | 사람 |
|------|------|
| 중요성은 아무리 강조해도 지나치지 않습니다 | (삭제) 왜 중요한지 한 가지 근거로 |
| ~은 필수불가결한 요소입니다 | ~이 없으면 X가 안 된다 (구체) |

## 3. 구두점 오용 — S1

| 슬롭 | 사람 |
|------|------|
| 라벨 — 설명 (em-dash) | 라벨 : 설명 (콜론) |
| 범위 5–10 (en-dash) | 5~10 |

## 4. 빈 수식어 — S2

뺐을 때 의미가 안 변하면 슬롭이다.

`혁신적인 · 획기적인 · 체계적인 · 종합적인 · 효과적인 · 핵심적인 역할`
→ 구체로 바꾸거나 삭제. ("혁신적인 방법" → "기존보다 절반 빠른 방법")

## 5. 기계적 연결어 — S2

| 슬롭 | 사람 |
|------|------|
| 결론적으로 / 요컨대 | (삭제) 결론을 그냥 말함 |
| 다음과 같습니다: | 콜론 나열 대신 문장으로, 정말 병렬일 때만 목록 |
| 아래와 같습니다: | 위와 동일 |

## 6. 헤지·완충 — S2

자신 없어 보이게 만드는 꼬리.

`~라고 할 수 있습니다 / ~라고 볼 수 있습니다 / ~것으로 보입니다 / ~것으로 생각됩니다`
→ 단정하거나(근거 있으면) 근거를 댄다. ("효과적이라고 할 수 있습니다" → "응답이 23% 빨라졌다")

## 7. 명사화·수동태 과잉 — S2

| 슬롭 | 사람 |
|------|------|
| ~에 대한 검토가 이루어졌다 | ~을 검토했다 |
| 개선이 필요한 상황입니다 | 개선해야 한다 |

동사를 명사로 굳히고 수동으로 빼면 글이 늘어지고 주체가 사라진다.

## 8. 영어 차용어 — S3

`레버리지 · 인사이트 · 퀄리티 · 디테일`(불필요한 차용) → 우리말로. (단, 정착된 용어 'API·UI'는 허용)

## 9. 목록 남발 — 구조

모든 걸 불릿으로 쪼개면 논리(인과·우선순위)가 사라진다. **3개 이하**, 그 외엔 산문으로 통합. 정말 병렬·체크리스트일 때만 목록.

## 10. 리듬 단조 — voice

문장 길이가 다 비슷하면 기계처럼 읽힌다. 짧은 문장과 긴 문장을 섞는다. 소리 내어 읽었을 때 숨이 차면 자른다.

## 등급 기준 (verify.sh)

- **A** : S1·S2·S3 모두 0 (발행 가능)
- **B** : S3만 남음
- **C** : S2 남음 (다듬기 권장)
- **D** : S1 잔존 (**게이트 실패**)

목표는 항상 **A**. S1은 타협 없이 0.
