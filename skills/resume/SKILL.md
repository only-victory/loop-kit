# /resume : 이력서·자기소개서 제작

채용 담당자의 첫 스캔을 통과하는 **단일 HTML 이력서**를 만든다. 파일 하나로 어디서든 열리고 그대로 인쇄(PDF)된다.

> 이 스킬의 목표는 "내가 한 일을 적는 것"이 아니라 **"읽는 사람이 6초 안에 '이 직무에 맞는 사람'이라고 느끼게 하는 것"**이다.

## 제1원칙 : 직무 적합성이 먼저다

이력서는 자서전이 아니다. **지원 직무가 요구하는 역량을, 내 경험으로 증명하는 문서다.** 같은 경력도 직무에 맞게 번역한다. 첫 화면(이름 아래 한 줄 요약 + 핵심 성과)에서 적합성이 보여야 한다.

## 제2원칙 : 활동이 아니라 성과다

"무엇을 담당했다"가 아니라 **"무엇을 해서 어떤 결과를 냈다"**를 쓴다. 모든 경력 항목에 가능한 한 수치를 붙인다(매출·전환율·기간 단축·규모). 수치가 성과를 증명한다.

## 절차 (이 순서를 지킨다)

1. **인터뷰** : 아래 [깊은 인터뷰]로 결·지원 직무·핵심 성과를 끌어낸다.
2. **직무 분석** : 지원 직무의 핵심 요구 역량 3\~5개를 먼저 정한다.
3. **한 줄 요약** : "OO년차 [직무], [핵심 역량]으로 [대표 성과]"를 한 줄로.
4. **제작** : 요구 역량에 맞춰 경력을 성과 중심으로 배치한다(단일 HTML).
5. **검증** : `scripts/verify.sh`로 구조·연락처·수치·구두점을 확인한다.

## 깊은 인터뷰 (옵션은 고르게, 내용은 받는다)

**0단계 : 이력 결 (선택 창으로 고르게 한다)**

- ① 경력 이력서결 : 한 줄 요약 → 경력(성과 중심) → 역량 → 학력 (경력직)
- ② 신입 자기소개서결 : 지원 동기 → 역량 근거(STAR) → 성장 가능성 → 학력·활동 (신입)
- ③ 포트폴리오형결 : 대표 프로젝트 3\~5개(문제·역할·결과) → 역량 → 약력 (직무 증명형)

**1단계 : 지원 직무·대상**

- **지원 직무·회사** : 구체적으로 (직무 요구 역량을 여기서 추출)
- **요구 역량** : 채용 공고에서 강조하는 것 3\~5개
- **차별점** : 나만의 강점 한 가지

**2단계 : 핵심 내용 (필수)**

- **대표 성과** : 수치로 증명되는 가장 강한 성과 2\~3개
- **경력·프로젝트** : 역할 + 행동 + 결과(수치)
- **역량·스킬** : 직무 관련 기술·도구

## 이력 결 골격 (HTML 단일 파일)

| 결 | 섹션 골격 |
|----|-----------|
| 경력 이력서결 | 헤더(이름·연락처·직무) → 한 줄 요약 → 핵심 성과 → 경력(성과·수치) → 역량 → 학력 |
| 신입 자기소개서결 | 헤더 → 지원 동기 → 역량 근거(STAR 일화) → 성장 의지 → 학력·대외활동 |
| 포트폴리오형결 | 헤더 → 한 줄 소개 → 대표 프로젝트(문제·역할·결과) → 역량 → 약력 |

**공통 규약:**
- 외부 CDN script 0개, 외부 CSS 0개 (자체완결), CSS는 `<style>` 인라인
- `@media print` 인쇄 대응 필수 (이력서는 인쇄·PDF 제출이 많다)
- 제목 `<h1>`(이름) 필수, 연락처(이메일·전화) 명시
- A4 1\~2장 기준, em-dash·en-dash 금지(구분자는 콜론)

## 안티패턴 (이게 보이면 다시 만든다)

- 직무와 무관한 경력을 시간순으로 나열 → 직무 적합성 순으로 재배치한다.
- "OO 업무 담당" 활동 나열 → "OO를 해서 OO 결과"로, 수치를 붙인다.
- 한 줄 요약이 없음 → 6초 스캔에서 핵심을 놓친다.
- 성과에 수치가 없음 → 증명되지 않은 주장은 약하다.
- 자기소개서가 추상적 다짐("열정을 다하겠습니다") → STAR로 구체 근거를 든다.
- 3장 이상 → 핵심만 1\~2장으로. 안 읽힌다.

## 검증 (필수 : loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <만든-이력서.html>
```

- [ ] (자동·HARD) 외부 CDN script/stylesheet 0건
- [ ] (자동·HARD) em-dash/en-dash 0건
- [ ] (자동·HARD) 제목 `<h1>`(이름) 존재
- [ ] (자동·WARN) 연락처(이메일·전화) 존재
- [ ] (자동·WARN) 경력·프로젝트 섹션 존재
- [ ] (자동·WARN) 역량·스킬 섹션 존재
- [ ] (자동·WARN) 정량 성과(숫자·%) 존재
- [ ] (자동·WARN) `@media print` 존재
- [ ] (육안) 6초 안에 직무 적합성이 보임
- [ ] (육안) 모든 경력이 성과·수치로 쓰임
- [ ] (육안) 지원 직무 요구 역량과 연결됨
