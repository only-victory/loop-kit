# /email : 업무 이메일 작성

읽는 사람의 시간을 아끼는 **명확한 업무 이메일**을 쓴다. 산출물은 바로 복사해 보낼 수 있는 제목 + 본문 텍스트다.

> 이 스킬의 목표는 "정중하게 쓰는 것"이 아니라 **"받는 사람이 첫 3문장 안에 용건을 알고, 마지막에 무엇을 할지 아는 것"**이다.

## 제1원칙 : 용건이 먼저다

인사와 배경으로 빌드업하지 않는다. **첫 문단에 "무엇을 / 언제까지 / 왜"가 보여야 한다.** 바쁜 사람은 첫 3줄로 열지 말지 정한다. 맥락·근거는 그 뒤에 받친다.

## 절차 (이 순서를 지킨다)

1. **인터뷰** : 아래 [깊은 인터뷰]로 결·상대·용건을 끌어낸다.
2. **한 줄 용건** : 본문을 쓰기 전에 "나는 [상대]에게 [무엇]을 [언제까지] 요청/보고한다"를 한 줄로.
3. **제목 확정** : 제목만 봐도 용건과 기한이 보이게 (예: "[요청] 7/3까지 계약서 검토 부탁드립니다").
4. **작성** : 용건 → 맥락 → 요청·다음 단계 → 맺음·서명.
5. **검증** : `scripts/verify.sh`로 제목·요청·서명·길이·구두점을 확인한다.

## 깊은 인터뷰 (옵션은 고르게, 내용은 받는다)

**0단계 : 이메일 결 (선택 창으로 고르게 한다)**

- ① 요청·의뢰결 : 무엇을·언제까지를 명확히, 상대 부담 낮추기 (가장 흔함)
- ② 보고·공유결 : 결론 먼저, 핵심 3개, 첨부·링크 정리
- ③ 조율·일정결 : 선택지 2\~3개 제시, 결정 쉽게
- ④ 사과·거절결 : 사실 인정·대안 제시, 완곡하되 모호하지 않게

**1단계 : 상대·관계 (어조를 정한다)**

- **받는 사람** : 상급자·임원 / 동료·팀 / 외부 고객·거래처 / 다수 수신
- **관계 격식** : 높음(처음·공식) / 보통(평소 협업) / 낮음(친한 팀)
- 회신으로 상대가 **무엇을 하길** 바라나 (마지막 문장이 이걸 명확히)

**2단계 : 핵심 내용 (필수)**

- **용건** : 한 줄로 (요청/보고/조율/사과)
- **기한·일정** : 언제까지 (없으면 "참고" 명시)
- **맥락** : 상대가 판단하는 데 필요한 최소 정보

## 이메일 결 (결마다 구조가 다름, 예절 규약은 공통)

| 결 | 본문 골격 |
|----|-----------|
| 요청·의뢰결 | 용건 1줄 → 배경(왜 필요) → 구체 요청(무엇·기한·형식) → 부담 완화(가능 여부 회신) → 맺음 |
| 보고·공유결 | 결론·핵심 → 상세 3개 → 첨부·링크 → 다음 단계 → 맺음 |
| 조율·일정결 | 용건 → 선택지 2\~3개(날짜·옵션) → 회신 요청 기한 → 맺음 |
| 사과·거절결 | 사실 인정 → 사유(간결) → 대안·다음 가능성 → 맺음 |

**공통 예절 규약:**
- 제목은 용건 + 기한이 보이게 (`[요청]`, `[공유]`, `[일정]` 머리표 권장)
- 한국 호칭·직함 존중 (OOO님, OO 팀장님), 과한 미사여구·읍소 금지
- em-dash·en-dash 금지, 구분자는 콜론(`:`)
- 한 화면 안에 끝나게 (스크롤 없이), 한 문단 3\~4줄

## 출력 형식

```
제목: [머리표] 핵심 용건 (기한)

(인사 1줄)
(용건 1\~2줄 : 무엇을 언제까지)
(맥락·근거)
(구체 요청 또는 다음 단계)
(맺음 1줄)

(서명 : 이름 / 소속 / 연락처)
```

## 안티패턴 (이게 보이면 다시 쓴다)

- 제목이 "안녕하세요" 또는 "문의" → 용건·기한이 안 보인다.
- 인사·날씨·근황 3줄 후 본론 → 용건이 먼저다.
- "바쁘시겠지만 죄송하지만 혹시 가능하시다면" 읍소 연쇄 → 한 번만, 명확히 요청.
- 요청에 기한이 없음 → 언제까지인지 없으면 우선순위가 밀린다.
- 한 메일에 용건 5개 → 핵심 1\~2개로, 나머지는 분리하거나 목록으로.
- 거절인데 "검토해보겠습니다"로 모호 → 거절은 완곡하되 결론은 분명히.

## 검증 (필수 : loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <만든-이메일.md>
```

- [ ] (자동·WARN) 제목 라인 존재
- [ ] (자동·WARN) 요청·다음 단계 단서 존재
- [ ] (자동·WARN) 서명·맺음 존재
- [ ] (자동·WARN) 분량이 한 화면 (과도하게 길지 않음)
- [ ] (육안) 첫 3줄에 용건이 보임
- [ ] (육안) 기한이 명확
