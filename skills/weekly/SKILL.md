# /weekly : 주간 업무 보고

상급자·팀이 1분 안에 한 주를 파악하는 **WBR(Weekly Business Review) 형식 주간 보고**를 쓴다. 산출물은 그대로 붙여넣을 수 있는 마크다운이다.

> 이 스킬의 목표는 "한 일을 나열하는 것"이 아니라 **"읽는 사람이 무엇을 알아야 하고, 나를 무엇으로 도와야 하는지 아는 것"**이다.

## 제1원칙 : 활동이 아니라 결과다

"무엇을 했다"가 아니라 **"무엇이 어떻게 되었다"**를 쓴다. 가능한 모든 항목에 수치를 붙인다(완료율·지표·금액·건수). 한 줄 요약을 맨 위에 둬서 바쁘면 그것만 읽어도 되게 한다.

## 절차 (이 순서를 지킨다)

1. **인터뷰** : 아래 [깊은 인터뷰]로 독자·기간·핵심 성과를 끌어낸다.
2. **한 줄 요약** : 보고를 쓰기 전에 이번 주를 한 줄로 ("핵심 성과 / 가장 큰 이슈").
3. **실적 정량화** : 한 일마다 결과·지표를 붙인다. 숫자가 없으면 상태(완료·진행 %)라도.
4. **작성** : 6섹션 골격에 채운다.
5. **검증** : `scripts/verify.sh`로 섹션·요약·지표·구두점을 확인한다.

## 깊은 인터뷰 (옵션은 고르게, 내용은 받는다)

**0단계 : 보고 밀도 (선택 창으로 고르게 한다)**

- ① 압축결 : 한 줄 요약 + 핵심 3개 + 이슈 (팀장·임원 빠른 확인)
- ② 표준결 : 6섹션 전체 (일반 주간 보고)
- ③ 상세결 : 6섹션 + 지표 표 + 근거 (월말·분기 누적 보고)

**1단계 : 독자·범위**

- **읽는 사람** : 팀장 / 본부장·임원 / 팀 공유 / 협업 부서
- **보고 기간** : 주차·날짜 범위
- 읽는 사람이 이 보고로 **결정하거나 도와야 할 것** (지원 요청을 정한다)

**2단계 : 핵심 내용 (필수)**

- **금주 실적** : 한 일 + 결과·지표 (3\~6개)
- **이슈·리스크** : 막힌 것, 도움이 필요한 것
- **차주 계획** : 다음 주 할 일 (우선순위순)

## WBR 6섹션 골격

```
# 주간 업무 보고 : (이름 / 팀 / 기간)

## 한 줄 요약
이번 주 핵심 성과 한 줄 + 가장 큰 이슈 한 줄.

## 금주 실적
- (한 일) : (결과·지표) 예) 신규 제안 3건 발송, 1건 미팅 확정
- ...

## 핵심 지표
- 지표명 : 이번 주 값 (전주 대비 ±)

## 이슈 · 리스크
- (이슈) : (영향) / (현재 조치)

## 차주 계획
- (우선순위순 할 일)

## 지원 요청
- (누구에게) (무엇을) (언제까지)
```

## 안티패턴 (이게 보이면 다시 쓴다)

- "OO 회의 참석, OO 작성" 활동 나열 → 결과·지표로 바꾼다.
- 수치가 하나도 없음 → 최소한 완료율·건수라도 붙인다.
- 이슈 섹션이 비어 있음 → 정말 없으면 "없음", 보통은 숨은 리스크가 있다.
- 지원 요청이 없음 → 막힌 게 있으면 명확히 요청한다(주간 보고의 핵심 기능).
- 한 줄 요약이 없음 → 바쁜 사람은 요약만 읽는다.
- 차주 계획이 10개 → 우선순위 3\~5개로.

## 검증 (필수 : loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <만든-주간보고.md>
```

- [ ] (자동·HARD) em-dash/en-dash 0건
- [ ] (자동·WARN) 한 줄 요약 섹션 존재
- [ ] (자동·WARN) 금주 실적 + 차주 계획 섹션 존재
- [ ] (자동·WARN) 이슈·리스크 섹션 존재
- [ ] (자동·WARN) 정량 지표(숫자·%) 존재
- [ ] (육안) 활동이 아니라 결과로 쓰였는가
- [ ] (육안) 1분 안에 한 주가 파악되는가
- [ ] (육안) 지원 요청이 구체적인가
