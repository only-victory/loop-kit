# /promo : 교육 홍보 페이지 제작

강의·특강·스터디·워크숍·아카데미 등 **교육 프로그램 홍보 웹페이지**를 단일 HTML 파일로 만든다.
신청 링크 연동, 모바일 대응, 외부 의존성 없음(폰트 CDN 제외).

> 이 스킬의 목표는 "정보를 나열하는 것"이 아니라 **"신청 버튼을 누르게 만드는 것"**이다.
> 예쁜 정보 100줄보다, 욕구를 자극하는 카피와 구조가 전환을 만든다.

## 절차 (이 순서를 지킨다)

1. **인터뷰** : 아래 [깊은 인터뷰]로 프로그램 정보를 먼저 끌어낸다. 주제만 듣고 만들지 않는다.
2. **섹션 설계** : 프로그램 유형에 맞는 섹션 조합을 결정한다 → `reference/sections.md`
3. **제작** : `templates/promo.html.txt`를 베이스로 내용을 채운다.
4. **검증** : `scripts/verify.sh`로 규약 위반을 막고, [육안 체크리스트]로 카피·전환을 본다.

## 깊은 인터뷰

> **선택지가 정해진 질문(테마·유형·커리큘럼 형식)은 반드시 선택 창으로 띄워** 고르게 한다.
> 사용자가 직접 입력해야 하는 건 **프로그램 내용·설명·링크뿐**이다.

**0단계 : 스타일 (선택 창으로 고르게 한다)**

- **테마** : ① 다크 골드(기본·세련) ② 라이트 블루(전문·신뢰) ③ 라이트 크림(따뜻·친근) ④ 네이비(비즈니스·격식)
- **프로그램 유형** : 단일 특강(1회) / 단기과정(2~4주) / 정규과정(5주+) / 스터디 / 워크숍
- **커리큘럼 형식** : 타임라인(날짜 중심) / 그리드(주차 압축) / 세로 카드(섹션 상세)

**1단계 : 핵심 정보 (필수)**
- 프로그램명과 한 줄 설명 (Hero 카피)
- 대상 / 모집 인원 / 기간 / 장소 / 신청 마감일
- Google Forms 또는 신청 링크 URL

**2단계 : 콘텐츠 (여기서 깊이가 갈린다)**
- 교육 목표 : 수강 후 무엇을 할 수 있나? (3개 이내 아웃컴)
- 커리큘럼 : 회차별 주제 + 형태(강의/실습/자기주도/과제)
- "이런 분께 추천" 페르소나 (2~4개)
- 수료 조건 (출석·과제·기타)

**3단계 : 차별화 (선택)**
- 강사 소개 (강사 기반 프로그램은 필수)
- 수치 강조 : 누적 수강생, 만족도, 재수강률 등
- 특별 혜택 : 수료증, 수료 후 지원, 네트워킹 기회

> 2단계를 건너뛰면 "정보 나열 페이지"가 나온다. 목표·커리큘럼·페르소나 세 가지는 꼭 채운다.

## 섹션 구성 원칙

**필수 섹션** (순서 유지):
`header` → `meta-chips` → `info-table(교육 개요)` → `curriculum` → `apply-box` → `footer`

**추가 섹션** (프로그램 특성에 맞게 삽입):
- `ticker` : 관련 키워드·업종 무한 스크롤 (시각적 임팩트용)
- `brief` : 한 문단 도입 설명 (복잡한 배경이 있을 때)
- `notice` : 중요 공지 (마감 임박, 선발 기준 등)
- `recommend-grid` : 추천 대상 (페르소나가 명확할 때)
- `goals` : 교육 목표 카드 (아웃컴이 3개 이상일 때)
- `how-block` : 운영 방식 설명 (독특한 구조일 때)
- `stats-bar` : 수치 강조 바 (실적 데이터가 있을 때)
- `instructor-card` : 강사 소개 (강사 신뢰가 중요할 때)
- `conditions` : 수료 조건 (이탈 방지, 기대치 조율)

상세 안내 : `reference/sections.md`

## 디자인 시스템

**폰트** : Pretendard (본문) + JetBrains Mono (레이블·배지·번호). CDN에서 로드.

**기본 색상 (다크 골드 테마)** — 테마 전환은 `reference/themes.md`의 `:root` 블록으로 교체:
```css
--bg:#0a0b0d  --surface:#15171c  --accent:#dfb839
--text-hi:#f3f2ed  --text-md:#d5d7df  --text-lo:#9fa1a8
```

**구두점** : em-dash(`—`)·en-dash(`–`) 금지. 구분자는 **콜론(`:`)**.

**CTA 전환 원칙** :
- 신청 버튼은 **최소 2회** 등장 (Hero 하단 또는 meta 직후 + 최하단 apply-box)
- 버튼 텍스트 : "신청하기" 보다 "지금 신청하기" / "마감 전 신청하기"가 전환율 높음
- 마감일을 버튼 근처에 함께 표시 (긴박감 유도)

## 안티패턴 (이게 보이면 다시 만든다)

- Hero 카피가 프로그램명 반복 → "왜 이 프로그램인가"를 한 문장으로
- 신청 버튼이 페이지 맨 아래 한 번만 → Hero 직후에도 추가
- 강사 소개 없음 (강사 기반) → 신뢰의 핵심, 반드시 포함
- 수료 조건 없음 → 신청자가 중도 이탈하면 양쪽 다 손해
- 모든 항목이 동일한 font-size → 시각 계층 없으면 읽지 않는다

## 검증 (필수 — loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <만든-페이지.html>
```

자동 검사(verify.sh) + 육안 체크리스트를 **둘 다** 통과해야 완료다.

- [ ] (자동·HARD) h1 프로그램 제목 존재
- [ ] (자동·HARD) 신청 버튼·링크 존재 (apply-btn 또는 forms 링크)
- [ ] (자동·HARD) footer(기관명·문의처) 존재
- [ ] (자동·HARD) em-dash/en-dash 0건
- [ ] (자동·HARD) 외부 JS CDN 0건
- [ ] (자동·WARN) 교육 개요 테이블 존재
- [ ] (자동·WARN) meta chips 존재
- [ ] (육안) Hero 카피가 "왜 이 프로그램인가"를 담음
- [ ] (육안) 신청 버튼이 최소 2회 등장
- [ ] (육안) 모바일에서 버튼이 탭 가능한 크기 (최소 44px)
- [ ] (육안) Google Forms 링크가 실제 작동
