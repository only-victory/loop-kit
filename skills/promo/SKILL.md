# /promo : 교육 홍보 페이지 제작

강의·특강·스터디·워크숍·아카데미 등 **교육 프로그램 홍보 웹페이지**를 단일 HTML 파일로 만든다. 신청 링크 연동, 모바일 대응, 외부 폰트 CDN만 허용.

> 이 스킬의 목표는 "정보를 나열하는 것"이 아니라 **"신청 버튼을 누르게 만드는 것"**이다. 예쁜 정보 100줄보다, 욕구를 자극하는 카피와 구조가 전환을 만든다.

## 제1원칙 : 결이 먼저다

페이지를 만들기 전에 "어떤 브랜드 인상이어야 하는가"를 확정한다. 결이 정해지면 폰트·색·여백·도형 언어가 따라온다. 결 없이 정보만 나열하면 기억에 남지 않는 페이지가 된다.

## 절차 (이 순서를 지킨다)

1. **인터뷰** : 아래 [깊은 인터뷰]로 결·목적·내용을 먼저 끌어낸다.
2. **섹션 설계** : 프로그램 유형에 맞는 섹션 조합을 결정한다 → `reference/sections.md`
3. **제작** : 고른 결의 `templates/promo.<결>.html.txt`를 복제해 채운다.
4. **검증** : `scripts/verify.sh`로 규약 위반을 막고, [육안 체크리스트]로 카피·전환을 본다.

## 깊은 인터뷰 (옵션은 고르게, 내용은 받는다)

선택지가 있는 건 **선택 창**으로 띄워 고르게 한다. 직접 받는 건 **프로그램 내용뿐**이다.

**0단계 : 디자인 결 + 프로그램 유형 (선택 창 필수)**

결은 색만이 아니라 **폰트·타이포 스케일·여백·도형·CTA 스타일**까지 통째로 정의한다. 색은 결 안에 포함되어 따로 고르지 않는다. 정의·토큰값은 `reference/design-languages.md`.

- **디자인 결** : 프로그램 브랜드에 맞는 것을 고른다
  - ① 프리미엄 다크 : 세리프 헤드라인·넓은 여백·금/크림 강조 (고급 브랜드·심화 과정)
  - ② 모던 클린 : 그로테스크·강한 그리드·블루 포인트 (기업 교육·테크 스쿨)
  - ③ 에너지 임팩트 : 빅 타이포·하이 콘트라스트·오렌지 (부트캠프·집중 과정)
  - ④ 소프트 아카데믹 : 둥근 카드·웜 크림 배경·그린/세이지 (학원·대학·정규 교육)
- **프로그램 유형** : 단일 특강(1회) / 단기과정(2~4주) / 정규과정(5주+) / 스터디 / 워크숍
- **커리큘럼 형식** : 타임라인(날짜 중심) / 그리드(주차 압축) / 세로 카드(섹션 상세)

**1단계 : 핵심 정보 (필수)**

- 프로그램명과 한 줄 설명 (Hero 카피)
- 대상 / 모집 인원 / 기간 / 장소 / 신청 마감일
- 신청 링크 URL

**2단계 : 콘텐츠 (여기서 깊이가 갈린다)**

- 교육 목표 : 수강 후 무엇을 할 수 있나 (3개 이내 아웃컴)
- 커리큘럼 : 회차별 주제 + 형태(강의/실습/자기주도/과제)
- "이런 분께 추천" 페르소나 (2~4개)
- 수료 조건 (출석·과제·기타)

**3단계 : 차별화 (선택)**

- 강사 소개 (강사 기반 프로그램은 필수)
- 수치 강조 : 누적 수강생·만족도·재수강률
- 특별 혜택 : 수료증·수료 후 지원·네트워킹

> 2단계를 건너뛰면 "정보 나열 페이지"가 나온다. 목표·커리큘럼·페르소나 세 가지는 꼭 채운다.

## 디자인 시스템 (결마다 인상이 다름, 섹션 어휘는 공통)

4개 결은 **같은 섹션 구조**를 쓰되 폰트·색·여백·도형 언어가 다르다. 섹션 배치 방법은 결과 무관하게 동일하고, 결을 바꾸면 같은 내용도 전혀 다른 인상이 된다.

**공통 섹션 어휘:**

| 섹션 | 역할 | 필수/선택 |
|------|------|-----------|
| `header` | 프로그램명·Hero 카피·주요 메타·신청 버튼 | 필수 |
| `meta-chips` | 기간·장소·인원·형식 칩 | 필수 |
| `info-table` | 교육 개요 표 | 필수 |
| `curriculum` | 커리큘럼 (타임라인/그리드/카드) | 필수 |
| `apply-box` | 신청 CTA (마감일·버튼) | 필수 |
| `footer` | 기관·문의처 | 필수 |
| `ticker` | 키워드 무한 스크롤 | 선택 |
| `recommend-grid` | 추천 대상 페르소나 | 선택 |
| `goals` | 교육 목표 카드 | 선택 |
| `stats-bar` | 수치 강조 바 | 선택 |
| `instructor-card` | 강사 소개 | 선택 |
| `conditions` | 수료 조건 | 선택 |

결별 상세·CSS : `reference/design-languages.md`

## CTA 전환 원칙

- 신청 버튼은 **최소 2회** 등장 (header 하단 + apply-box)
- 버튼 텍스트 : "신청하기"보다 "지금 신청하기" / "마감 전 신청하기"가 전환율 높음
- 마감일을 버튼 근처에 함께 표시 (긴박감 유도)

## 안티패턴 (이게 보이면 다시 만든다)

- 결 선택 없이 기본 템플릿 그대로 → 결부터 골라야 브랜드 인상이 생긴다
- Hero 카피가 프로그램명 반복 → "왜 이 프로그램인가"를 한 문장으로
- 신청 버튼이 페이지 맨 아래 한 번만 → header 직후에도 추가
- 강사 소개 없음 (강사 기반) → 신뢰의 핵심, 반드시 포함
- 수료 조건 없음 → 신청자가 중도 이탈하면 양쪽 다 손해
- 모든 항목이 동일한 font-size → 시각 계층 없으면 읽지 않는다
- 색을 결 밖에서 임의로 바꿈 → 결의 팔레트를 그대로 쓴다

## 검증 (필수 : loop-kit 게이트)

```bash
bash ${CLAUDE_SKILL_DIR}/scripts/verify.sh <만든-페이지.html>
```

자동 검사(verify.sh) + 육안 체크리스트를 **둘 다** 통과해야 완료다.

- [ ] (자동·HARD) `<h1>` 프로그램 제목 존재
- [ ] (자동·HARD) 신청 버튼·링크 존재
- [ ] (자동·HARD) `<footer` 요소 존재
- [ ] (자동·HARD) em-dash/en-dash 0건
- [ ] (자동·HARD) 외부 JS CDN 0건
- [ ] (자동·WARN) 신청 버튼 2회 이상 등장
- [ ] (자동·WARN) 교육 개요 테이블 존재
- [ ] (자동·WARN) meta chips 존재
- [ ] (육안) Hero 카피가 "왜 이 프로그램인가"를 담음
- [ ] (육안) 결의 인상이 프로그램 성격과 맞음
- [ ] (육안) 모바일에서 버튼이 탭 가능한 크기 (최소 44px)
- [ ] (육안) 신청 링크가 실제 작동
